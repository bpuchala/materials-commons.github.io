File operations¶

Overview¶

Typical filesystem operations can be performed with mc ls, mc mkdir, mc rm, and mc mv. By default, these act on both local and remote files and directories. If you want them to act on only remote files or directories use the --remote-only option.

The mc ls command displays a table showing information on both local (prepended with “l_”) and remote files and directories (prepended with “r_”). Including the --checksum option will calculate the MD5 checksum of the local file for comparison with the remote file. The result of the comparison is shown in the “eq” column. For example, checking contents of both the current (“.”) and the “level_1” directory:

$ mc ls . level_1 --checksum
.:
l_updated_at          l_size    l_type     r_updated_at          r_size    r_type     eq    name        id
--------------------  --------  ---------  --------------------  --------  ---------  ----  ----------  -------
2020 Aug 17 23:25:48  23B       file       2020 Aug 18 03:59:05  23B       file       True  file_A.txt  2659322
2020 Aug 17 23:25:48  23B       file       -                     -         -          -     file_B.txt  -
2020 Aug 17 23:25:48  -         directory  2020 Aug 18 03:31:46  0B        directory  -     level_1     2659316

level_1:
l_updated_at          l_size    l_type     r_updated_at          r_size    r_type     eq    name             id
--------------------  --------  ---------  --------------------  --------  ---------  ----  ----------  -------
2020 Aug 17 23:25:48  31B       file       2020 Aug 18 03:31:47  31B       file       True  file_A.txt  2659317
2020 Aug 17 23:25:48  31B       file       2020 Aug 18 03:31:49  31B       file       True  file_B.txt  2659318
2020 Aug 17 23:25:48  -         directory  2020 Aug 18 03:31:49  0B        directory  -     level_2     2659319

mc ls --help documentation¶

List local & remote directory contents

usage: mc ls [-h] [--checksum] [--json] [--dataset DATASET_ID] [--include] [--exclude] [--clear] [paths [paths ...]]

Positional Arguments¶

paths

Files or directories

Default: [‘/Users/bpuchala/Work/codes/mccli’]

Named Arguments¶

--checksum

Calculate MD5 checksum for local files

Default: False

--json

Print JSON exactly

Default: False

--dataset

Specify a dataset to act on.

Default: “”

--include

Include files and directories in the specified dataset. Including a directory includes all files and sub-directories recursively, unless prevented by –exclude-files.

Default: False

--exclude

Exclude file and directories from the specified dataset. Excluding prevents inclusion of files and directories that would otherwise be included due to a higher-level directory being included.

Default: False

--clear

Clear files and directories from the include/exclude selection lists of the specified dataset. A file or directory may still be included in or excluded from the dataset afterwards if a higher level directory is included or excluded

Default: False

mc mkdir --help documentation¶

Make remote directories

usage: mc mkdir [-h] [-p] [--remote-only] [paths [paths ...]]

Positional Arguments¶

paths

Directory names

Default: [‘/Users/bpuchala/Work/codes/mccli’]

Named Arguments¶

-p

Create intermediate directories as necessary

Default: False

--remote-only

Make remote directories only. Does not compare to local tree.

Default: False

mc rm --help documentation¶

remove files and directories

usage: mc rm [-h] [-r] [--remote-only] [--no-compare] [paths [paths ...]]

Positional Arguments¶

paths

Files or directories

Named Arguments¶

-r, --recursive

Remove recursively

Default: False

--remote-only

Remove remote files only. Does not compare to local files.

Default: False

--no-compare

Remove even if local and remote files differ.

Default: False

mc mv --help documentation¶

Move files. Use mc mv <src> <target> to move and/or rename a file or directory. Use mc mv <src> … <directory> to move a list of files or directories into an existing directory.

usage: mc mv [-h] [--remote-only] [paths [paths ...]]

Positional Arguments¶

paths

Sources and target or directory destination

Named Arguments¶

--remote-only

Move remote files only. Does not compare to local files.

Default: False
